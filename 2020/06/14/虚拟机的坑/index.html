    安装 Ubuntu

    安装 Ubuntu 的时候切记不要选择推荐项目

    主要是这一步，如果选择了第二项的话，它会在安装之前让你设置账户密码什么的，如果填写的话，Ubuntu 进去就直接到这个界面，你么得选择

    可以点 Skip，这一步是安装语言包什么的，但是我第一次安装的时候，点了 Skip 直接卡在一个黑屏界面了… 所以直接选第三项安装就完事了，然后进入安装阶段，选择语言，时区什么的，最后也对到上图这个界面，再点 Skip 就没事。

    再然后就是修改一下镜像源，可以通过选择 系统设置 -> 软件和更新 选择下载服务器 -> "mirrors.aliyun.com" ，也可以手动修改。

    打开 /etc/apt/sources.list 然后全部代替为（以下内容直接拷贝来的，放心使用）

    ubuntu 20.4 的配置：（其他自行查阅 https://developer.aliyun.com/mirror/ubuntu?spm=a2c6h.13651102.0.0.4ee41b11IEUSBW）

    deb http://mirrors.aliyun.com/ubuntu/ focal main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal main restricted universe multiverse

deb http://mirrors.aliyun.com/ubuntu/ focal-security main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal-security main restricted universe multiverse

deb http://mirrors.aliyun.com/ubuntu/ focal-updates main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal-updates main restricted universe multiverse

deb http://mirrors.aliyun.com/ubuntu/ focal-proposed main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal-proposed main restricted universe multiverse

deb http://mirrors.aliyun.com/ubuntu/ focal-backports main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal-backports main restricted universe multiverse

    安装 VMWare Tools

    这玩意简单来说就是让你用的更舒服，一般在安装系统的时候就会自动提示让安装上，如果错过了的话，进入系统之后可以点击 虚拟机 --> 安装 VMWare Tools 来实现。在桌面会出现一个新的光碟图标的文件，双击安装即可。

    设置 Git 用户名，密码，秘钥

    执行下面两条命令

    git config --global user.name "xxx"

git config --global user.email "xxx@xxx.com"

    之后生成 SSH 秘钥

    ssh-keygen -t rsa -C "xxx@xxx.com"

    安装 SSH 服务端和客户端

    sudo apt-get install openssh-client 
sudo apt-get install openssh-server

    然后要修改一个配置文件，这样可以从外部来连接，修改 PermitRootLogin 和 PasswordAuthentication

    sudo nano /etc/ssh/sshd_config

    还要注意一点，如果要准备去连接的电脑上有之前的记录，记得把记录删除。

    Ubuntu

    这块是真的坑死我… 什么样的问题都让我碰到了，什么1396 Error，1035 Error，1698 Error啥的全让我碰到了… 最开始是通过 apt 安装… 但是就是少配置这个步骤… debian.cnf 文件里根本就没有初始的账号密码… 试过添加 skip-grant-table 发现也不行…然后卸了重装MySQL两次，第二次操作失误导致 MySQL 无法启动… 之后就直接重新装了虚拟机… 折腾了一溜十三招。最后用 wget 直接去下个包，通过 dpkg 来安装，然后执行 apt update 和 apt upgrade 命令更新 apt，再执行 apt install mysql-server 和 spt install mysql-client

    （这次是通过外部主机 SSH 来安装的）然后然后然后… 坑又来了… 还是没出现让配置默认密码这个界面，直接就让选密码强度了… 我真是裂了… 然后这回 debian.cnf 文件就有数据了，用 cat 查看到初始给的账号密码，是可以正常登陆的，但是谁会每次都敲这么复杂的密码… 但是作为备份还是不错的，我就考虑留下，然后自己在创一个管理员权限的用户。创建用户这也遇到坑了… 直接留正确的操作顺序吧… 显示创建用户，虽然还没开始学 MySQL，但是 @ 后应该表示一个域，这个似乎可以换成 % 代表全部列表

    create user 'root'@'localhost' identified by '111111';

    创建之后刷新一次，避免奇奇怪怪的坑，在用户/用户权限这面，操作一次刷新一次准没错…

    flush privileges;

    之后就是赋权了，第一个星号我估计应该是代表所有数据库的意思吧

    grant all privileges on *.* to 'root'@'localhost';

    再然后，注意一点，注意一点，注意一点，删除用户的话，先执行 Delete 语句，然后刷新，再再再执行 drop 语句应该最稳妥… 一开始我直接 drop 是不行的… 但是 Delete 可以，然后 Delete 之后在列表里是看不到了… 但是似乎好像还残留什么东西… 没学不懂… 还需要再执行 drop 语句

    Delete FROM user where user='root' and Host='localhost';
flush privileges;
drop user root@'localhost';

    最后！修改密码的正解在这里，以及 MySQL 版本，原型环境和加密协议：

    alter user 'root'@'localhost' identified by '666666';
-- 8.0.20 MySQL Community Server - GPL
-- Ubuntu 20.04 LTS
-- MySQL 8.0 加密规则

    Windows

    啥也不说了，直接上正解，注意全称要用管路员模式！

    https://www.bilibili.com/video/BV1VE411x7fZ

    第一步，下载安装包，解压缩

    第二步，设置环境变量，提前创好文件夹，执行 mysqld --intialize --console 和 mysqld -install mysql 命令，初始化+添加服务

    第三步，启动服务 net start mysql

    第四步，用随机给的密码进入之后，修改密码

    ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_nativa_password BY '111111';

    最后 flush privileges; 结束
